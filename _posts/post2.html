Semi-Manual Play-list Auto Audacity Audio Recording Script

by Kirk.O

Posted on October 22, 2019 at 11:30 PM

My second "practical" problem solving program/script that I made, also in AutoHotKey. This one I have used more than the previous, just because I have more use for it than the other, this one is generally less complex though.

I made this script, because I found from using another program to save my music play-list from Youtube was not working for many of the songs. The reason (I believe) that I found is that Youtube does not save the audio file sources on every video, so that you can't access the source audio in a neat little downloadable file format at you whim. So what I decided was probably the best work around for this was to just record the audio through something like Audacity and have the "source" file and be able to do with it as a please, legal or not.

Now, I had various ideas on how I could go about doing this in a script. The first was to be nearly fully automated, to some extent. I would bascially just have the program wait for a period of silence (such as when the song ended) and use that to determine when to save the current audio file and do the steps to go onto the next and do that for the entire play-list. The problem with this I quickly found was that it was a very error prone process to accurately determine what was the actual end of the song with this method.

The way I was monitoring the audio was using this handy audio library that was made by an AutoHotKey community member, that basically used the same functionality as the Windows Vista Audio library, but it was accessible through this library, very handy tool. After doing a lot of tweeking I made a fairly consistent method, this method basically just looped constantly and tracked the peak audio volume being processed at that second, and was constantly checking if that level went below a threshold with a simple if-statement inside this loop. When the audio went low enough, the rest of the script ran and did the necessary steps to save the current file and start the process over again for the next song.

Now this whole thing works very well in theory, but in actual practice, there are a lot of things that easily throw off this entire process and basically breaks the script if you happen to look away from it long enough, which was initially the plan, run the script before I went to sleep and wake up to a folder full of neatly saved audio files. The problem was that things such as video buffering would throw the whole thing out of wack and cause incomplete audio files and many missed songs, another thing was songs with ocasional periods of low volume would do the same, finally the biggest problem was videos that had little to no period of silence at the end, this would completely mess up the script in multiple ways. The primary issue was that if videos had no silence, the script would continue to the next video, before properly counting that the previous one was finished, and everything would get out of wack and would usually lead to one giant recording instead of the individual files I wanted.

The first thing I tried was taking out the silence detection all together (which made things less fancy is some sense), and the way I was tracking the video ending now was through some web-scrapping of the HTML code that would find the start time and end times of the video, and do a small equation that would convert these into a "Sleep command" duration in seconds, this would in theory end the video at the exact time the video SHOULD be about to end minus 1 or 2 seconds. Like the previous issue, this one had problems too, the biggest one that I was not initially expecting was that the "Sleep" command was extremely inaccurate when the times become large enough, such as anything over 1-2 minutes long, and the accuracy would depend completely on what the current processing load was on the PC, so if I was doing stuff while running this, the time would be +-6-11 seconds or so, which was not going to work. After I found this out, I just decided to do a semi-manual set-up that I have now until I make a more complex version with more checks and balances to be as autonomous as possible. For now though, the manual version that takes out a lot of the tedious works for my purposes, I hope to revise this eventually though and polish it up.
